CSV Watcher

An automated file import system that monitors network folders and loads SmartCare tool data directly into SQL Server — completely eliminating the problematic SSIS pipeline that caused chronic data delays and duplicates for SOC Internal Dasboard.

Python Flask SQL Server Windows Service ETL Huawei SmartCare ODBC
// The Problem

SOC data pipeline relied on SSIS (SQL Server Integration Services) to load SmartCare tool exports into SQL Server. SSIS was notoriously fragile — it caused frequent data delays, silently produced duplicate records, and required constant manual intervention. The team had no visibility into what was happening or why imports were failing.

// The Solution

CSV Watcher replaces SSIS entirely with a lightweight Python service that watches designated network folders, detects new SmartCare CSV exports by filename keyword, and appends data to SQL Server transactionally. With a live dashboard, import heatmap, and job scheduler — the team now has full visibility and zero data delays.

// how it works
Data Flow
01

SmartCare exports CSV

Huawei SmartCare automatically generates CSV files containing network KPI data and drops them into designated FTP/network folders on the server.

02

Windows Service detects new file

The CSV Watcher Windows Service polls configured watch folders every few seconds. When a new file appears matching the keyword pattern, it triggers the import process.

03

Data appended to SQL Server

The file is read, cleaned (NUL characters removed), and appended transactionally to the target SQL Server table using ODBC. Row counts are logged at every step.

04

File archived or rejected

Successfully imported files move to the Archive folder. Files that don't match any watcher keyword are moved to Rejected — nothing is silently dropped.

05

Dashboard & Jobs confirm success

The live log updates in real time. The Jobs page shows a green/red heatmap per watcher — daily, hourly, or minutely — so the team can instantly see if any data source missed its window.

// features
Key Features
Live Log Dashboard
Real-time log viewer auto-refreshing every 5 seconds. Shows every file event, import progress, row counts, and errors as they happen.
Watcher Management
Add, edit, enable, or disable folder-to-table mappings from the UI. Each watcher defines a watch folder, keyword filter, target server, database, and table.
Import Heatmap
Jobs page shows a color-coded heatmap of import success per watcher — green for OK, red for missing. Granularity: daily, hourly, or minutely.
In-App Table Creator
Upload a CSV to auto-detect column names and data types, or define columns manually. Generates and executes the CREATE TABLE SQL directly on the target server.
Multi-Server Support
Connect multiple SQL Server profiles with connection testing built in. Each watcher can target a different server independently.
Zero Duplicates
Each file is processed once and immediately archived. No retry loops, no double-processing — the exact problem that plagued the old SSIS setup.
