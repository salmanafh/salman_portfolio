SmartCare Helper

An LLM-powered chatbot with RAG (Retrieval-Augmented Generation) that helps engineers use Huawei SmartCare more effectively — from querying data and building dashboards to identifying network anomalies on XLSMART's network.

Python LLM RAG Ollama Vector DB NLP Huawei SmartCare
// The Problem

Huawei SmartCare is a powerful but complex platform. New team members spend weeks learning how to query the right data, build meaningful dashboards, and interpret network KPIs. Experienced engineers also lose time re-looking up procedures for anomaly identification — knowledge that lives in scattered documents and tribal know-how.

// The Solution

SmartCare Helper is a RAG-based chatbot that ingests internal SmartCare documentation as its knowledge base. Engineers can ask questions in natural language — "how do I query 5G drop rate by site?" or "what does a sudden PDCP throughput drop indicate?" — and get accurate, context-aware answers instantly.

// features
Key Features
RAG Knowledge Base
Answers grounded in actual SmartCare documentation — query guides, dashboard tutorials, and XLSMART-specific anomaly identification procedures.
Source Citations
Every answer references the specific document and section it was retrieved from — engineers can verify and trace answers back to the original source.
Multi-Model Support
Switch between Qwen2.5, LLaMA 3.1, GPT-OSS, and Gemma3 depending on accuracy needs, speed requirements, or network availability.
Runs Locally
Primary models run via Ollama on local infrastructure — sensitive telecom data never leaves the internal network, keeping compliance intact.
Natural Language Interface
Engineers ask questions the way they think — no need to know exact menu paths or counter names. The chatbot maps intent to the right SmartCare procedure.
Onboarding Accelerator
New team members can get up to speed on SmartCare in days instead of weeks — the chatbot serves as an always-available internal expert.
