Auto Demarcation

An end-to-end data pipeline that ingests KPI drop alarms from Huawei SmartCare and automatically identifies the root cause contributor — whether it's a site, MME, PGW/GGSN, or a specific application — eliminating the need for manual network investigation.

Python SNMP Data Pipeline Huawei SmartCare KPI Analysis Bash / Shell Script Linux
// The Problem

When a KPI drop alarm fires in XLSMART's network, engineers had to manually investigate the source — cross-checking data across sites, MME nodes, PGW/GGSN, and application layers. This process was slow, inconsistent, and dependent on individual expertise, often delaying response time for critical network issues.

// The Solution

Auto Demarcation takes a SmartCare alarm file as input and runs it through an automated analysis pipeline that cross-references KPI data across all network layers. Within seconds, it produces a structured output identifying the exact contributor — site, MME, PGW/GGSN, or app — so engineers can act immediately.

// features
Key Features
Alarm Event Dashboard
Displays all KPI drop alarms from SmartCare with trigger time, alarm type, GGSN/Province, and MME. Supports search, pagination, and filtering across thousands of events.
One-Click Download
Users simply click Download on any alarm row. The backend automatically processes the alarm data and returns a structured contributor report — no manual steps needed.
Multi-Layer Analysis
The pipeline cross-checks KPI data across all network layers simultaneously — site, MME, PGW/GGSN, and application — to pinpoint exactly where the degradation originated.
Dark / Light Mode
Full theme toggle built into the app — engineers can switch between dark and light mode based on their working environment and preference.
Alarm Filtering
Filter by alarm type across 20 unique categories. Search by alarm name, GGSN, or province to quickly surface the events that matter most.
Real-Time Data
System Online indicator confirms live connectivity to SmartCare. The dashboard reflects current alarm data — 55 trigger events and 17,314 total records loaded in production.
