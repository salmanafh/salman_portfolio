Traffic Forecasting
Dashboard

A predictive traffic forecasting system for XLSMART's network — using ARIMA/SARIMA on historical SmartCare KPI data to project future load trends, visualized in an interactive Power BI dashboard for proactive capacity planning.

Python ARIMA / SARIMA Power BI Apache Spark SmartCare Time Series SQL
// The Problem

XLSMART's network capacity planning is largely reactive — congestion and throughput degradation are identified only after they've already impacted users. There's no systematic way to anticipate traffic surges, seasonal patterns, or long-term load growth across sites, MMEs, and PGW nodes before they become critical issues.

// The Solution

By applying ARIMA/SARIMA time series models to historical SmartCare KPI data, this project will produce short and long-term traffic forecasts per network element. Results are published to a Power BI dashboard that gives the operations team a forward-looking view of network load — enabling proactive capacity decisions before congestion hits.

// model
ARIMA / SARIMA Approach
// step 01
Stationarity Check
Run ADF test on traffic time series. Apply differencing if non-stationary to remove trend and make the series suitable for ARIMA modeling.
ADF · d-differencing
// step 02
ACF / PACF Analysis
Identify AR (p) and MA (q) orders from autocorrelation and partial autocorrelation plots. Detect seasonal period S from data patterns.
ACF · PACF · S=24
// step 03
SARIMA Fitting
Fit seasonal ARIMA to capture both short-term autocorrelation and recurring daily/weekly traffic patterns typical in telecom networks.
SARIMA(p,d,q)(P,D,Q,S)
// step 04
Evaluation & CI
Evaluate model with MAE and RMSE on holdout set. Generate forecast with 95% confidence intervals to communicate prediction uncertainty.
MAE · RMSE · 95% CI
// features
Planned Features
7 / 30-Day Forecast
Short and medium-term traffic projections per region, site, and network element — giving the operations team actionable planning horizons.
Congestion Risk Flag
Automatically flag network elements where forecasted traffic exceeds defined capacity thresholds — turning predictions into actionable alerts.
Drill-Down by Node
Power BI slicers to filter forecast by region, site, MME, or PGW — so teams can zoom from network-wide view to individual node level.
Scheduled Refresh
Pipeline runs on a schedule — new SmartCare data feeds the model automatically, keeping forecasts current without manual intervention.
Seasonal Awareness
SARIMA captures daily and weekly traffic cycles inherent to telecom data — peak hours, weekend patterns, and recurring load signatures.
Export Ready
Forecast outputs stored in SQL Server — accessible to other tools and pipelines beyond Power BI, including Auto Demarcation context enrichment.
